Information We Do NOT Collect

Owl.md does NOT collect, store, or transmit:

  • Personal information (name, email, phone number, etc.)
  • User accounts or credentials
  • Location data
  • Device identifiers
  • Usage statistics or analytics
  • Crash reports or diagnostics
  • Any personally identifiable information (PII)

Data That Stays on Your Device

The following data is stored locally on your device only and never leaves your device:

1. Document Content

  • Markdown files you create or edit
  • PDF files you generate
  • All document content remains on your device

2. App Preferences

  • Theme preference (light/dark mode)
  • Selected PDF theme
  • App settings

Stored using: Android SharedPreferences (local storage)

3. File History

  • List of recently opened files (file paths only)
  • Accessed dates of files

Stored using: Android SharedPreferences (local storage)

4. Announcement Read Status

  • Which in-app announcements you've viewed
  • Timestamps of when announcements were read

Stored using: Android SharedPreferences (local storage)

Note: All of this data is stored locally on your device. We have no access to it, and it is never transmitted to our servers or any third party.

Network Requests

Owl.md makes limited network requests in the following scenarios:

1. GitHub Markdown Loading (Optional Feature)

  • When: You explicitly choose to load a markdown file from a GitHub URL
  • What: HTTP request to fetch the raw markdown content from GitHub
  • Data Sent: The GitHub URL you provide
  • Recipient: GitHub.com
  • Purpose: To download the markdown file content you requested

2. In-App Announcements (Optional Feature)

  • When: App checks for announcements (every 4 hours when online)
  • What: HTTP request to fetch announcements.json from our GitHub repository
  • URL: https://raw.githubusercontent.com/drmafouad/Owl.md/main/announcements.json
  • Data Sent: None (simple GET request)
  • Purpose: To inform you of app updates, new features, or important notices

Important: These network requests do not include any personal information, do not track your identity or device, and are made to public GitHub repositories only.

Permissions

Storage Permissions

Permissions: READ_EXTERNAL_STORAGE, WRITE_EXTERNAL_STORAGE (Android 12 and below)
READ_MEDIA_IMAGES, READ_MEDIA_VIDEO, READ_MEDIA_AUDIO (Android 13+)

Purpose: To allow you to:

  • Load markdown files from your device storage
  • Save PDF files to your device storage
  • Access files you choose through the file picker

What we access: Only the specific files you explicitly select through the system file picker. We do not scan, index, or access any other files on your device.

Internet Permission

Permission: INTERNET

Purpose: To enable optional features like loading markdown files from GitHub URLs and fetching in-app announcements.

Note: All core features (editing, PDF generation, local file operations) work completely offline.

Data Retention and Deletion
